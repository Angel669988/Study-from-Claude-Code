1. 为什么 AI PM 要黑客级学习 Claude Code

你真正会获得什么

  • 把“体验问题”拆成可归因的系统问题：Prompt、Runtime、Tools、Governance、Cost 哪层出问题一眼可判。
  • 把“优化想法”转成可实验杠杆：一次只改一层，效果可验证。
  • 把“版本迭代”变成外环系统：每周有差距表、有优先级、有回归验收。

不学会发生什么

  • 只能凭直觉调 Prompt，无法解释为什么成功率波动。
  • “提升”不可复现，团队争论大于决策。
  • 需求越多，系统熵越高，成本和风险一起上升。

AI PM 黑客级学习心法

  • 看源码不是为了写 Rust，而是为了拿到“决策杠杆图”。
  • 每读一层都回答三个问题：这层解决什么业务问题？能调什么？如何验收？
  • 一切结论必须回到指标：一次完成率、回合数、误拦截率、成本。

一句话

  • Claude Code 对 PM 的价值：它把“Agent 产品”从玄学调参，变成可运营工程。

2. 学习地图：从产品问题倒推源码

方法：先定义目标函数，再看源码层杠杆，最后做实验闭环。

2.1 主链路：输入 -> 迭代 -> 执行 -> 治理 -> 沉淀

CLI 入口分流-> Prompt 组装上下文-> run_turn 循环-> 工具执行-> Permission/Hook 闸门-> Usage/Session 沉淀
对应源码：main.rs + prompt.rs + conversation.rs + tools/lib.rs + permissions.rs/hooks.rs + usage.rs/session.rs

2.2 目标函数：你作为 PM 要赢什么

成功率
一次完成率（无需二次追问）
效率
平均回合数 / 工具调用数
风险
高风险工具误放行率
成本
单任务 token 成本

3. 七大可学能力：你能学到什么

3.1 Prompt 上下文工程

🛠️ 源码事实

  • 分层构建 system prompt：系统规则、执行准则、环境上下文、项目上下文、CLAUDE 指令、运行配置。
  • 自动发现并注入 CLAUDE.md/.claude/instructions.md，并做去重+预算截断。

你能学到

  • 如何把“产品策略”从口头要求变成可执行上下文。

图形化说明

系统规则-> 项目指令-> 去重/预算-> 最终 Prompt
例子：你要做“中文 query 命中优化”，把“改写只允许追加，不覆盖原句”写入项目指令后，模型输出更稳定，减少策略漂移。
上下文工程策略注入

📈 PM 可用法

  • 把关键业务规则写成项目指令文件，而非放在会议纪要里。
  • 做 Prompt 变更灰度：只改一段，观测一次完成率变化。
验收：A/B 比较“改前 vs 改后”30 条 query，一次完成率上升且平均回合数不升。

3.2 Runtime 迭代闭环

🛠️ 源码事实

  • run_turn 是循环系统：模型输出 tool_use -> 执行 -> 回流 -> 再决策 -> 收敛。
  • 内置停机条件、最大迭代、自动压缩阈值与累计 usage。

你能学到

  • 如何设计“可持续交付”而非“一次性回答”的 Agent 产品。

图形化说明

用户问题-> 模型决策-> 工具执行-> 结果回流-> 停机输出
例子：“帮我修 bug”通常要先读日志再改代码，run_turn 允许多轮工具调用并在条件满足时收敛，而不是第一轮硬答。
迭代系统收敛策略

📈 PM 可用法

  • 重点追“平均回合数”和“失败后恢复率”，这是一次完成率的前置指标。
验收：看“失败后恢复率”是否提升，避免把失败从第 1 轮推迟到第 3 轮。

3.3 Tool 路由与执行

🛠️ 源码事实

  • 工具面有注册清单、输入 schema、权限要求、统一执行入口。
  • CLI 入口可做 allowed_tools 规范化与白名单限制。

你能学到

  • 如何把能力扩展做成“注册式平台”，避免主链路被污染。

图形化说明

工具注册-> schema 校验-> 权限检查-> 执行路由
例子：“查文件+改代码”会拆成读文件工具与写入工具，命中正确工具链才可能一次完成。
工具平台路由质量

📈 PM 可用法

  • 把“命中正确工具”作为中间指标，驱动 rewrite/路由优化。
验收：先看“首轮工具命中率”，再看最终一次完成率，定位问题更快。

3.4 Permission/Hook 治理

🛠️ 源码事实

  • 权限模式+升级审批路径是独立策略层。
  • Pre/Post Hook 在工具前后可拦截、反馈并改变后续决策。

你能学到

  • 如何把“合规风控”嵌入执行闭环而不是事后补救。

图形化说明

模型提议动作-> Permission 判定-> Pre Hook 拦截-> 执行/拒绝-> Post Hook 记录
例子：高风险命令默认拦截并要求确认，系统会把拒绝信息回给模型，触发更安全的替代方案。
风险控制可审计

📈 PM 可用法

  • 主指标提升必须挂 3 个护栏：误拦截率、误放行率、时延。
验收：看“危险动作被拦住”同时“正常任务不被误伤”，而不是只看拦截数量。

3.5 Memory 与指令继承

🛠️ 源码事实

  • 项目记忆来自指令文件层级发现与注入，不是黑盒向量库。
  • 存在可查看路径（如 memory 报告）来解释“为什么这么做”。

你能学到

  • 如何把团队经验“资产化”，降低新人冷启动成本。

图形化说明

组织规范-> 项目指令文件-> 会话注入-> 输出一致
例子：把“先给结论再给步骤”的答复规范写入记忆层，团队不同成员跑出的文档风格更一致。
组织记忆一致性

📈 PM 可用法

  • 建立“规则更新 -> 回归测试 -> 指标验证”的记忆治理流程。
验收：抽样比较“新成员任务”与“老成员任务”的成功率差距是否收敛。

3.6 Cost/Compaction 成本控制

🛠️ 源码事实

  • usage 持续追踪输入/输出/cache token 与成本估算。
  • 上下文过长触发自动压缩，保留近期关键消息继续执行。

你能学到

  • 如何把成本控制嵌入运行时，而不是财务月末复盘。

图形化说明

usage 计量-> 接近阈值-> 自动压缩-> 继续任务
例子：长对话任务中，系统在接近 token 上限时自动 compact，避免中途崩掉重来。
成本治理长会话

📈 PM 可用法

  • 一次完成率优化必须同步看单任务成本，避免“贵成功”。
验收：压缩后任务成功率不降，同时单任务成本下降或持平。

3.7 Harness/Parity 外环演进

🛠️ 源码事实

  • 兼容层可抽取上游 commands/tools/bootstrap 快照。
  • parity audit 输出已实现/缺失覆盖比，支持版本化补齐。

你能学到

  • 如何把“追上游能力”从人肉对比升级为可重复审计流程。

图形化说明

上游快照-> 能力映射-> parity 审计-> Top3 补齐
例子：每周看“已实现/缺失/部分实现”比例，只把 Top3 高业务价值差距排进下个版本。
外环进化能力对齐

📈 PM 可用法

  • 每周固定跑一次差距审计，只放 Top3 差距进下版本。
验收：版本复盘必须回答“这次补齐是否推动主指标”。

4. 怎么用：AI PM 业务落地 Playbook

场景：把“一次完成率”设为主目标，结合 Claude Code 源码基座，做可归因增长。

7 天实验法（可直接执行）

  • Day1：抽取 30~100 条真实 query，建立基线（一次完成率/回合数/成本/误拦截）。
  • Day2-4：只改一个杠杆（如 query rewrite 或 permission 分级）。
  • Day5：回放同一批 query，看主指标是否提升且护栏不劣化。
  • Day6：沉淀失败样本库与规则修订单。
  • Day7：版本评审，只回答“哪个杠杆推动了目标函数”。

验收口径

  • 主指标：一次完成率上升。
  • 护栏：误拦截率不升、单任务成本不升、平均回合数不升。

你可以直接拿去开会的模板

  • 目标函数：成功率 / 效率 / 风险 / 成本。
  • 杠杆映射：Prompt / Runtime / ToolRoute / Governance / Cost。
  • 实验设计：样本、对照、观察窗、验收阈值。
  • 外环治理：parity 差距表 + Top3 排期。

5. 实战 Lab：真实 Query 与改写策略

5.1 离线可复现实验（已验证）

示例对比：原始中文 query 与补充能力词后的 query 在命中模块上会出现明显变化（例如更聚焦 conversation/hooks/permissions）。

5.2 Query Rewrite 规则（PM 版）

规则

  • 保留用户原句，不覆盖，只追加能力词。
  • 能力词来自“目标函数杠杆”而非同义词堆砌。
  • 命中失败时触发二次改写兜底。

目标

  • 提升一次完成率，不增加成本/回合/误拦截。

建议先追 3 个指标

  • 中文 query 首次命中率。
  • 0 命中后二次改写挽救率。
  • 改写后误命中率（命中但无业务价值）。
复现实验命令建议：cargo run ... system-prompt、python3 -m src.main route "..."、python3 -m src.main bootstrap "..."
