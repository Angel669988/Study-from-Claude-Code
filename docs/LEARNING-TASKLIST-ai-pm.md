## 1. Harness Engineering（第一优先）

- 学什么：把“提示词优化”升级成“端到端系统优化”（目标函数、实验、回归、发布节奏）。
- 你要能做：定义主指标（一次完成率）+ 护栏（成本/风险/时延）+ 每周实验循环。
- 为什么：这是从“会调”到“会经营”Agent产品的分水岭。

## 2. Query Rewrite 产品化

- 学什么：中文 query 的重写策略、失败兜底、命中率提升机制。
- 你要能做：原句保留、能力词追加、0命中二次改写、误命中控制。
- 为什么：它直接影响首轮工具命中和一次完成率。

## 3. 工具路由与能力编排（Tooling Surface）

- 学什么：工具注册、schema、权限级别、路由策略、失败恢复。
- 你要能做：把“功能点”变成“能力平台”，并建立工具命中漏斗指标。
- 为什么：主流 Agent 框架都在往“工具+MCP+多工具协同”走。
- 状态：[ ] 未开始  [ ] 进行中  [ ] 已完成

## 4. Runtime 收敛设计

- 学什么：run-turn 循环、停止条件、重试与恢复、自动压缩触发策略。
- 你要能做：区分“该继续迭代”vs“该停机交付”的产品规则。
- 为什么：决定用户体感是“聪明执行”还是“瞎转圈”。
- 状态：[ ] 未开始  [ ] 进行中  [ ] 已完成

## 5. Memory/Instruction 治理

- 学什么：记忆写入策略、冲突治理、版本化回滚、记忆质量评估。
- 你要能做：把团队经验变成可审计、可回归的系统资产。
- 为什么：没有记忆治理，规模一大就会策略漂移。
- 状态：[ ] 未开始  [ ] 进行中  [ ] 已完成

## 6. Hook/Permission 安全闭环

- 学什么：前后置拦截、风险分级、误拦截/误放行平衡。
- 你要能做：把安全从“阻断团队”变成“可量化治理”。
- 为什么：企业化落地里，安全和效率必须一起被优化。
- 状态：[ ] 未开始  [ ] 进行中  [ ] 已完成

## 7. 可观测性与评测体系（Eval + Observability）

- 学什么：会话级指标、任务级指标、离线回放、线上监控。
- 你要能做：建立“问题定位树”（是 Prompt、路由、权限还是成本导致）。
- 为什么：没有可观测，就无法稳定迭代。
- 状态：[ ] 未开始  [ ] 进行中  [ ] 已完成

## 8. Agent-First 研发流程（组织能力）

- 学什么：文档化上下文、可读代码库、计划工件化、PR 吞吐治理。
- 你要能做：把团队流程设计成“AI可读、AI可执行、AI可验证”。
- 为什么：这是未来 1-2 年 AI 团队效率差距的核心来源。

## 逐周推进方式

- 每周只攻克 1 项，不并行开太多战线。
- 每项都做：源码研读 -> 实验设计 -> 指标验收 -> 复盘沉淀。
- 每周复盘只回答一句话：
  - “这周这项能力，是否推动了一次完成率提升，且护栏不劣化？”
