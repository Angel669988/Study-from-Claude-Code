# AI 大模型产品经理黑客学习版（Claude Code 源码学习）

作者：十月

## 这份文档解决什么问题

这不是一份“Rust 代码讲解”，而是一份 **AI PM 可直接落地的学习与经营手册**：

- 如何从 Claude Code 源码里提炼产品杠杆。
- 如何用“一次完成率”组织实验，而不是凭感觉调 Prompt。
- 如何把 Agent 产品做成“可治理、可回归、可演进”的工程系统。

## 核心学习主线

文档围绕一条可执行主链路展开：

`输入 -> 上下文组装 -> 迭代执行 -> 工具路由 -> 权限治理 -> 成本控制 -> 外环演进`

对应到 PM 的目标函数：

- 成功率：一次完成率（用户无需二次追问）
- 效率：平均回合数 / 工具调用数
- 风险：误放行率 / 误拦截率
- 成本：单任务 token 成本

## 七大能力模块（PM 读法）

1. Prompt 上下文工程：把策略从口头要求变成可执行约束
2. Runtime 迭代闭环：把一次回答升级为可收敛执行系统
3. Tool 路由与执行：把能力扩展做成注册式平台
4. Permission / Hook 治理：把风控放到执行前后闭环里
5. Memory 与指令继承：把团队经验资产化
6. Cost / Compaction：把成本控制内置到运行时
7. Harness / Parity：把“跟上游”做成持续外环

## 文档亮点

- 每个模块都有：
  - 图形化流程说明
  - PM 场景例子（输入 -> 动作 -> 结果）
  - 验收口径（如何证明优化有效）

## 推荐使用方式

1. 先看总览和目标函数，统一评估语言。
2. 每周只选一个杠杆做实验（避免多变量互相污染）。
3. 每次复盘必须回答一句话：
   - “这次变化是否提升了一次完成率，且护栏不劣化？”
